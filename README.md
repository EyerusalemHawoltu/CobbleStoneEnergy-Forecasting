# CobbleStoneEnergy Power Forecasting Pipeline

**German Day-Ahead Power Price Forecasting → Prompt Curve Translation**

End-to-end prototype with:
- **FastAPI backend** — data ingestion, QA, LightGBM forecasting, DA→curve translation
- **React frontend** — chat interface powered by a Groq AI agent (free Llama 3.3 70B)
- **Groq LLM** (free) — AI agent with 6 tools + QA rule proposer + daily commentary
- Demo mode works with zero API keys (synthetic data)

## Setup

### 1. Python Environment

Requires Python 3.11+.

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. API Keys

```bash
cp .env.example .env
# Edit .env with your credentials
```

| Key | Required? | Where to get it |
|---|---|---|
| `GROQ_API_KEY` | Strongly recommended (free) | console.groq.com — free tier, no credit card |
| `ENTSOE_API_KEY` | Optional | transparency.entsoe.eu → My Account → Web API Security Token |

> Without `GROQ_API_KEY`: AI agent and commentary are disabled. Without `ENTSOE_API_KEY`: backend runs in **demo mode** with synthetic data (still fully functional for the UI).

## Running the Web App (Backend + Frontend)

### Terminal 1 — FastAPI backend

```bash
# From project root
uvicorn backend.main:app --reload --port 8000
```

The backend auto-initialises the pipeline on startup (~30s). Open the interactive API docs at http://localhost:8000/docs.

### Terminal 2 — React frontend

```bash
cd frontend
npm run dev
```

Open http://localhost:5173 in your browser. The UI polls the backend every 2 seconds until ready, then enables the chat interface.

### What you can ask the AI agent

```
"What's the forecast for 2024-01-15?"
"Show me the monthly delivery curve"
"How accurate is the LightGBM model compared to the baseline?"
"What's the current trading signal?"
"Run data quality checks on the dataset"
"Write a market commentary for tomorrow"
"What's the expected peak price next week?"
```

The Groq agent (Llama 3.3 70B, free tier) interprets the question, calls the right pipeline tool, and returns both a natural-language answer and structured data that the frontend charts automatically.

## Running the CLI Pipeline

### Full pipeline (downloads data, trains models, generates figures + submission.csv)

```bash
python pipeline.py \
  --start 2022-01-01 \
  --end 2024-12-31 \
  --test-start 2024-07-01
```

### Without AI features (no Anthropic key needed)

```bash
python pipeline.py --no-llm
```

### Using cached data only (no ENTSO-E key needed)

```bash
python pipeline.py --skip-download
```

### Force re-download (ignore cache)

```bash
python pipeline.py --force-refresh
```

## Key Design Decisions

### Market: Germany (DE)
Largest European power market; ENTSO-E data is complete and high-quality; strong
wind/solar penetration creates price dynamics relevant to VRE-driven analysis.

### Forecasting Target: Hourly DA prices (Option A)
Directly maps to the DA auction output. Delivery-period averages (week, month)
are derived by aggregating the hourly forecast — preserving shape information
that a direct baseload forecast would discard.

### No-leakage guarantee
All price lag features use ≥24h lookback. Fundamentals use lagged actuals
shifted 24h. The walk-forward CV trains strictly before the prediction window.

### Walk-forward CV (expanding window)
Mirrors real trading constraints. Re-trains every 7 days with expanding history.
Minimum training window: 180 days.

### LLM integration
Two components: (1) QA rule proposer — Claude generates pandas validation
conditions from schema + stats; (2) Daily commentary — Claude narrates computed
metrics in plain English. Both are fully logged (prompt, response, latency,
tokens) and fail gracefully.

## Data Sources (ENTSO-E Transparency API)

Base URL: `https://web-api.tp.entsoe.eu/api`  
Authentication: `?securityToken=<ENTSOE_API_KEY>`  
Full API guide: https://transparency.entsoe.eu/content/static_content/Static%20content/web%20api/Guide.html

| Dataset | documentType | Additional params |
|---|---|---|
| DA Prices | A44 | `in_Domain`, `out_Domain` = DE bidding zone |
| Total Load | A65 | `processType=A16`, `outBiddingZone_Domain` |
| Wind Onshore | A69 | `processType=A16`, `psrType=B18` |
| Wind Offshore | A69 | `processType=A16`, `psrType=B19` |
| Solar PV | A69 | `processType=A16`, `psrType=B16` |

German bidding zone EIC: `10Y1001A1001A63L`

## Submission CSV Format

`submission.csv` contains out-of-sample predictions for the test window (2024-07-01 → 2024-12-31):

```
id,y_pred
2024-07-01 00:00:00,72.45
2024-07-01 01:00:00,68.12
...
```

- `id`: UTC timestamp (YYYY-MM-DD HH:MM:SS)
- `y_pred`: Forecasted DA price in EUR/MWh
